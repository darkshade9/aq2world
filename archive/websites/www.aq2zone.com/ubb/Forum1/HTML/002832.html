Capo 		posted 10-24-2000 07:14 AM            
::sigh::

In Quake 2, id Software has some hard-coded limits in their renderer for hardware mode. In software mode, either the limit for the draw distance is much higher or limitless, I don't know for certain.

For the 3D rendering system, id Software hardcoded a limit on rendering distance. Remember that at the time Voodoo2 was the fastest 3D card around. Since they wanted a larger market, they had to program for cards that sucked a lot more. Rendition Verite V1000, Voodoo Rush, Diamond Edge3D (nVidia's NV1 baby), etc.

These cards did not have enough texture memory to handle scenes that were too large. Compound that with the fact that most systems didn't have enough RAM to handle all of the textures and the sounds and the game data when the scene got too large, and you'll understand why they put that limit in the game engine itself.

Yeah, it'd be nice if there were a variable or menu option you could set, like, "set gl_limit_draw_distance 0", but there ain't and you're out of luck.

If you happen to be insane, you can use the Rendition 3D/software renderer to run in software mode and then have a Rendition Verite V1000 apply filtering on each texture. It looks nice but it's slower than a really damn slow thing.

Perhaps it is a combination of the rendering distance and a limit on the skybox. The engine stops rendering at a certain limit and you get the HOM effect that we all know and love. And to hide that, the skybox is defaulted to a distance lower than the render distance, even at the extreme rotated edge of the sky. I can see how this would be an issue since there's at least one Quake 2 map that has a rotating sky. Though it doesn't really run up against any limits due to map size, I can see Carmack or some programmer think about this as a possible issue (skybox extending past the render limit) and make sure the distances match up in all cases.

- Q2 OpenGL hits rendering limit, and caps houses (partially) beyond a certain distance
- Q2 software doesn't have a rendering limit but a maximum triangles limit (and drops triangles beyond a certain number)

- HL hits a rendering limit softw and OpenGL
- the map HL rendering distance parameter is ignored
