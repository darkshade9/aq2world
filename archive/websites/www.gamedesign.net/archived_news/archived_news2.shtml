  • My Thoughts on QOOLE 99
    • I must say I am very impressed with QOOLE 99. It was a breeze to setup, even for Half-Life. The Texture Browser kicks all other editors asses, simple because it sorts the textures by name, and it doesnt load all the textures at the same time. So you can view all the wood textures by selecting the wood category, etc.

    Creating the brushes was the only problem I have with QOOLE. It's the same thing I didnt like about it when I gave it a try a few months back. I don't like having to change to a different mode to scale, rotate, move, etc. QERadiant makes scaling, creating, and rotating brushes much faster and easier, though nothing beats BSP in this category.

    Getting the map to save, compile, and run in the game is one area where QOOLE shines. Not once did I have a problem. My first test half-life map saved, compiled, and ran in Half-Life with ease. Which is perfect for beginners.

    Though I still like QERadiant better because of it's speed, QOOLE is definitly my second favorite editor available currently. I still need to take a better look at Quest and QuArK though. Havnt tried the newest versions yet. On an ending note...BSP would still be my favorite if it was still be updated. =)

    I started a thread on the MISC Discussion board about QOOLE 99. Post your thoughts there.

  • Winbspc v0.9 released
    • That awesome program that can decompile BSP files has just gone up in version number with many more enhancements. Here is the p00p straight from their webpage:

      Winbspc v0.9 released ...

      It is getting better and better. Now you can convert maps from all your favorite games.

      Some major improvements:

      WinBSPC now converts maps of the following games:

        - Quake 1
        - Quake 2
        - Sin
        - Half-Life


      - It converts .BSP or .MAP files into .AAS files that are needed to play the Gladiator bot.

      - It converts .BSP files into .MAP files. Handy if you lost your original .MAP file or you want to turn particular brushes in a certain .BSP file into a prefab.

      - It extracts files from .PAK or .SIN files.

      - It can read .BSP files directly out of .PAK or .SIN files.

      What's new since the last release:
      - Added conversion properties dialog.
      - Added Sin support.
      - Added Half-Life support.
      - Added Half-Life brush texturing.
      - Added Quake1 support.
      - Added extracting files from a Pak File.
      - Added file filter in Pak File dialog.
      - Added files sizes in Pak File dialog.


      What's been fixed:
      - Fixed sewer64.bsp overflow bug.
      - Fixed blocked conversion thread after properties were shown.
      - Fixed overwrite bug.
