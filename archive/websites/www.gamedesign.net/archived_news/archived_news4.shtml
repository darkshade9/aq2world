  • New MapSpy Released
    • A new version of that kickass map diagnostic utility has been released. Here is the email:

      Mad Dog and I have just released MapSpy v0.7. You can read more about it and download it from MapSpy's new home.

      If you're completely unfamiliar with MapSpy, it's a map diagnostic utility that traps errors in an uncompiled Quake2 .MAP file. MapSpy traps the most common qbsp3 errors and runtime errors, and typically provides much better diagnostic info than the compilers or game do - which hopefully will allow you to easily find and fix the problem *before* going to the trouble of compiling your map.

      Changes to this version:

      Added a buttload (i.e. a lot) of entity runtime error traps (e.g. missing req'd values, missing targets, invalid values, etc).

      A new "Find entities" feature that allows you to search for entities using several different criteria.

      Improved output for several errors. For example, "FloatPlane: bad normal" errors now tell you which brush caused the error.

      Floating-point support for editors that write floats to .map files (e.g. Quark)

      As always, we welcome comments, suggestions, and truly obnoxious criticisms.

      David Hyde


    So ummm...you heard the man. Go download it. I have also mirrored it here on the Rust File Server.

  • Short Tim Wright Q&A
    • Word is that a new version of ArghRad is being worked on. So I fired a few questions Tim Wright's way and asked him about it. Here is the short Q&A:

      What are the new improvements/features that we will be seeing in the new arghrad?

        ArghRad 2.0 will have phong shading, liquid/sky lighting tweaks, a plugin special-effects system, and proper Heretic2 support.

        The phong shading is done. Very cool for simulating curved surfaces! It's very configurable too, settable on individual faces.

        The lighting tweaks are done as well. These do a great job of eliminating the "splotches" and "black-band" problems often seen on faces near/touching sky and liquid faces.

        I'm working on the plugin system now. Neil White III (of Wally fame) will be helping create the initial set of effects.


      Do you ever plan on making a version of arghrad for other games?(halflife, sin, etc)

        Heretic2 support will be built into the next version. I have not yet made plans to support other games, however I may decide to do so in the future.


      When might we see the new version?

        I hope to release 2.0 in a couple weeks. The initial set of plugins may be released as a separate zip shortly after.


    Here is an comparison image of the new Arghrad below:(Click to enlarge)

  • Bobby Pavlock (aka Xcalibur) Interview
    • We got something very cool for yall today. Just posted an Interview with Bobby Pavlock (aka Xcalibur), Level Designer on Ion Storm's upcoming FPS action game Daikatana. Here is a teaser:

      How is Daikatana editing compared with other Quake/Quake2 engine based games? Any new functionality?

      Well, it is based on the Q2 engine, so it is very similar. It has alot of the same basic entities used in Q2 editing. Some of the cool new stuff we have is effects like fog, rain, snow, lightning, rotating trains, moveable glass and midtextures, parent targeting (so you can attach a button directly onto an elevator or lift and it travels with it), entities with sound flags built-in eliminating the need for 1000 target_speakers, etc. There's alot more, but I can't divulge everything :)
