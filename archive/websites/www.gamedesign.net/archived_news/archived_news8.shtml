 Tribes Mission Editor Update (3:00) Updated by [ weasl ] 

The new Tribes patch (1.3.12) includes some enhancments and fixes for the Mission Builder. Mission Editor Changes:
- New mission wizard for creating new missions
- Ability to scene light the current mission from within the editor
- Ability to move and edit mission static lights
- Random terrain generation from within the editor
- User configurable terrain, mission and world types for use in the mission wizard
- Script based landscape generation
- Ability to choose and place terrain textures
- Auto regeneration of command map after leaving editor
- Hotkey support for terrain editor
- Ability to add custom volumes from within the editor
- Ability to create new terrain types through scripts
- Can create a mission or enter the mission editor from the command line
 You can download the patch from here.

 Kingpin Editing (7:21) Updated by [ Fishman ] 

I saw a post on the gangland.org forums by Dan Koppel, level designer, working on Kingpin answering a question about making levels for the demo. Here is what he said:

    Yes, You can make maps that you can load and run in the demo. To do this you would need a couple of things. First a map editor that supports tga's ... The current release of QeRadiant supports tga's (by selecting hi-color textures) ... BSP's can be generated from maps with your favorate Q2 bsp, vis, and rad utilities. These bsp's can be run in the demo ... get the console by adding "+set developer 1" in your kingpin command line. KpRadiant (our version of radiant) will be released with the game and it does include many enhancements that are not in the current version QeRadiant.


Sounds cool...btw..the first user made map for Kingpin has been released and can be downloaded here
