    Updates contact RUST


    Preface:

    Here's a bunch of KP frequently asked questions. Including mapping, gameplay and common questions.


    MAPPING

    When I try to compile a map, the DOS screen flashes fast and closes.
    To compile your map you need to copy kpbsp.exe, kprad.exe and kpvis.exe into Kingpin/main/maps folder. You can find those files in the Kingpin/kprad folder.

    When I set texture light flag in kprad's Surface Inspector, no matter what the color of texture light is always white.
    When Xatrix was working on KP, they didn't have time to update ArgRad, and since KP based on Quake2 engine, it used wal files. After finishing KP, Xatrix recompiled textures to tga format, but forgot about wal lighting. You need to search the net for Kingpin wal textures.

    I made Deathmatch/Bagman map and when I try to load it, it loads in SinglePlayer mode.
    To load DM/BM map, you need to type in console "deathmatch 1", or start game with shortcut parameter "+deathmatch 1".

    If I want to check the map design, do I have to wait all the rad and vis time?
    You can pick "Bsp | bsp_only (novis/norad)" in kprad toolbar. It'll take less than a minute to compile, but you won't see lights and shadows in your map, it'll be fullbright.

    If I created entity, is it possible to undo it?
    Select your entity and pick "Selection | Ungroup entity" in kprad toolbar.

    Is it possible to preview how func_door, func_train, etc works in game?
    Select your entity and press "Ctrl-A", it won't work for specific KP entities, only Quake2 entities.

    When I hire guys in my map they can't follow me.
    You need to have navdata file in navdata folder. Load your map and type in console "nav_debug 1" and "nav_dynamic 1", now run everywhere you want your hired guys to follow. When you're done quit the game and navdata file will save itself in navdata folder.

    GAMEPLAY

    When I shoot at func_explosive or blow up characters, my KP crashes.
    You need to install the latest KP patch - v1.21.

    How can you zoom in KP?
    You need to use "fov" - field of view. fov 90 is the default fov. fov 180 is the highest fov. The smaller the number the more you zoom. Some players like to play with fov 120 cause you can see alot more than with the default fov.

    What's doublejump?
    Doublejump is a kind of jump. To do it, you need to find something small (less then 64 * 64 brush), then run toward it and before touching it, press jump twice fast. If you did it correctly you should jump higher than normal jump.

    What's bunnyhopping?
    Bunnyhopping is another kind of jump. To do it, you should press right strafe and turn your mouse 45 degrees to the right side, then press jump, keep jumping and cirle, you'll gain speed. You can use right or left strafe to bh or you can use them in combination. Don't press forward, the bunnyhop won't work, use only left or right strafe.

    Is there a weapon against bazooka?
    Flamethrower is a great weapon against bazooka, flame kills rockets, so if you aim at the enemy's bazooka he'll blow himself up.

    COMMON

    What voices Cypress Hill did in KP?
    They did voices of bartender, Momo and Jesus.

    What things didn't make into final version of KP?
    There was different kinds of mods for weapons. Three kinds of armor, chemical, flamable and bulletproof. Bumpmapping and drivable car, etc.

    Was there any movies, which inspired KP?
    Some of KP dialog was inspired by Pulp Fiction and Big Lebowski.

Console Commands

    CONSOLE COMMANDS

    To enable the ability to bring up the console during play
    add "+set developer 1" to the command line in you
    windows shortcut.

    CHEATS

    immortal - god mode
    give all - gives every item, weapon, etc.
    notarget - enemies can't see you.

    MAPPING

    r_speeds 1(0) - shows r_speeds and fps.
    gl_showtris 1(0) - shows polygons, won't work if you don't
    have the full OpenGL support graphics card.
    nav_debug 1(0) - debug the map's navdata.
    nav_dynamic 1(0) - enables to make navdata.

    OTHER

    noclip - fly through walls.
    map xxx - load map (xxx.bsp or xxx), load demo
    (xxx.dm2) or picture (xxx.tga), where xxx is the name of the map.
    exec xxx - execute config, where xxx is the config name.
    fov xxx - field of view, where xxx is the number. Default fov is 90.
    record xxx - record demo, where xxx is the demo name.
    stop - stop recording demo.
    togglecam - toggle chase-cam mode.

Compiling Maps In KPRadiant

    Compiling Maps In KPRadiant

    - by Zirco

    Introduction:

    Compiling is probably the most important thing you must learn in mapping. This tutorial will show you how to compile your map to bsp format in QERadiant.

    Step 1:

    If you have Kingpin installed to a custom directory or on a different hard drive letter:

    Step 2:

    By this time, you should have a map with a light entity in it. If not build one now. Remember, if your map does not have any light entities or light brushes in it, all you will see is dark when it comes to play your map.

    Save your map to .../Kingpin/main/maps, the directory may not exist so you will have to make a new folder called 'maps' in your 'main' folder. Next save your map to the .../Kingpin/KPRad directory when it comes time to compile and test your map. It is important that both maps are the exact same thing. Now go to the 'bsp' menu and click 'bsp_FullVis'. A DOS window should pop up then everything should run okay, if you get any errors and the compile process stops, it means your project file settings is not correct.

    Although it may seem useless and a little time consuming to save your map to two different locations, it is required for it to work.

    Step 3:

    When the compile process is done, close the window and QERadiant if you want then run Kingpin. Type in the console "map x" without quotation marks and x being the name of your map. If you do not have the console enabled, startup shortcut with "+set developer 1" without the quotation marks.

    Step 4 (optional):

    Other BSP compile types: If 'bsp_FullVis' takes too long or if you're map is huge, then try 'bsp_FastVis' or if you want to see your map with full brightness which takes only a second to compile then try 'bsp_only (novis/norad)'. When releasing a final map, use bsp_FullVis.

How to create character witch cigar, fedora hat, etc.?

    COUNT

    by Method

    Count defines character items and is used to assign a charatcter with different hat's and cigar's


    The count defines four types of characters items and is set with the enitity inspector. A count value of zero will assign no character item. The four types of character items are:

    1: Cigar,
    2: Fedora hat,
    4: Stetson hat; and
    8: Cap.

    Example 1:
    Open entity inspector for any selected character and set

    Key count
    Value 9

    The character entity will be assigned both a cap (8) and cigar(1). Where a value of 9 = 8 + 1.

    Example 2:
    Open entity inspector for any selected character and set

    Key count
    Value 5

    The character entity will be assigned both a stetson hat (4) and cigar (1). Where a value of 5 = 4 + 1.

    Example 3:
    Open entity inspector for any selected character and set

    Key count
    Value 2

    The character entity will be just assigned a Fedora hat and will not have a cigar (poor guy!!!)

How to create cutscene?

    How to create cutscene?

    by Method

    This is an example of how to use a camera to be targeted from a misc_cutscene_trigger.

    Misc_cutscene_trigger Options:

    targetname - camera target ID.

    cameraorigin - X Y Z camera start position.

    cameraangle - X Y Z start angles.

    rotate - X Y Z rotational velocity during cut scene.

    cameravel - [forward] [right] [up] speed to move from initial angle.

    cameravelrel - [forward] [right] [up] speed to move relative to current frame angle.

    wait - cut scene length in seconds (default 5).

    target - next camera to target.

    target2 - [NOT WORKING!!] camera angle points to this entity (overides other angle commands).

    deadticks - fov for this camera.

    duration - fade out time.

    reactdelay - time into camera to start fading out.

    Example:

    Create misc_cutscene_trigger and open Entity inspector.

    Key duration
    Value 1

     Value is the fade out time.

    Key target
    Value cam_1

     Value is the targetname of misc_cutscene_camera.

    Create misc_cutscene_camera and open Entity inspector.

    Key targetname
    Value cam_1

     Value is the camera target ID.

    Key name
    Value cut_scene/cut1.wav

     Value is the sound name to play.

    Key cameraangle
    Value 52 35 0

     Value is the X Y Z start angles.

    Key cameraorigin
    Value -2071 2534 263

     Value is the X Y Z camera start position.

    Key cameravel
    Value 7 50 -48

     Value is the [forward] [right] [up] speed to move from
    initial angle.

    Key cameravelrel
    Value 0 10 0

     Value is the [forward] [right] [up] speed to move relative
    to current frame angle.

    Key reactdelay
    Value 5

     Value is the time into camera to start fading out.

    Key wait
    Value 15

     Value is the cut scene length in seconds (default 5).
    Key target
    Value cam_2

     Value is the next camera to target.

    Key target2
    Value cam_1_viewtarget

     Value is the targetname of path_corner_cast.

    Key delay
    Value 5

     Value is the time into camera to start fading out.

    Key accel
    Value 0.3

     Value is the camera acceleration.

    Key decel
    Value 0.3

     Value is the camera deceleration.

    Create info_player_start and open Entity inspector.

    Key angle
    Value 225

     Value is the opening direction.

    Place info_player_start inside misc_cutscene_trigger.

    Create path_corner_cast and open Entity inspector.

    Key targetname
    Value cam_1_viewtarget

     Value is the path_corner_cast target ID.

    I used sr1.map cutscene as an example.

Setting Up QERadiant

    SETTING UP QERADIANT

    - by Zirco

    Introduction: This tutorial will help you to setup QERadiant that was
    installed with Kingpin.

    Step 1:

    If you have Kingpin installed to its default directory (c:\program files\Kingpin): With it installed to it's default directory, only the preferences should be modified, and not the project file which points everything to its correct places. First, startup QERadiant, it should ask you for the project file, locate kingpin.qe4 in Kingpin/KPRadiant then click 'open'.

    If you have Kingpin installed to a custom directory or on a different hard drive letter: With it installed to a foreign directory, you will need to change the Kingpin project settings and the QERadiant preferences. First, startup QERadiant, it should ask you for the project file, locate kingpin.qe4 in Kingpin/KPRadiant then click 'open'. This is the file we must modify, but to make it easy, we will do it within QERadiant. Go to the 'File' menu then click the 'Project Settings' submenu. Everything should be pointing to the default directories (c:\program files\kingpin...). Change the paths so they match the location you installed Kingpin to.

    Step 2:

    Go to the 'Edit' menu then click the 'Preferences' submenu. This will load up the common settings for QERadiant. Though some fields are not filled in, they are not required or needed.

    Step 3 (optional):

    Color Settings:

    When you have finished steps 1 and 2, you should by now have noticed the black grid on white background. It helps to change the colors to white grid on black background. We will use preset colors by going to the 'Misc' menu, 'Colors' submenu, then 'Themes', and finally click on 'Black and Green'.

    You will notice that it has made the minor grid disappear. This is because it's actually now black, so change it by going to the 'Misc' menu, 'Colors' submenu, then 'Grid Minor' to a dark gray color of your choice, I recommend the RGB to be 25,25,25.
