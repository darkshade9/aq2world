env_global

This entity is used to change the global state variable when triggered. You must trigger this entity with some type of trigger. If you set your "triggermode" to toggle, it will toggle the global state.

All global states default to off, so if you need it to be on, set the "initialstate" to "on". When the env_global is triggered, the global state will be changed to the "triggermode".

A common use of this entity is to have a button in one map trigger something in another map. To set this up you first need two maps with a button in one map that triggers something in another map.

To make this entity work, you need to first create it and set it up to do what you want, remembering to give it a targetname. Now you want to make your button trigger the env_global by giving the button a "target" that matches the "targetname" of the env_global.

Next we need to put a multisource entity in the map with the door. For the "globalname" field of the mutlisource, place the same value you have for the "globalstate" of the env_global. This allows the env_global to find the multisource entity. Lastly, we need to define a "target" for the multisource entity, which will be the "globalname" of our door.

That's it! Now the button in Map1 will open that door in Map2.

Valid Keys/Values and Flags
    keyvalues
    targetname - name
    globalstate - globalname of multisource
    triggermode - trigger mode
      0 = off
      1 = on
      2 = dead
      3 = toggle

    initialstate
      0 = off
      1 = on
      2 = dead

    flags
      set initial state (1) - This must be enabled for the initial state to be set. This is here mostly to allow global entities to be killed by the trigger without being affected by some initial state.
