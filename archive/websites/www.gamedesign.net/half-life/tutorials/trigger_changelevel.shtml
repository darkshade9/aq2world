trigger_changelevel

This entity is used to switch from one level to another. Also, info_landmark and trigger_transition are described in this tutorial because these entities all work together.

There are two types of level changes that can occur, seamless and not seamless.

To create a transition that is not seamless, like in Quake where you end up in a different room at the start of a new level, simply set the "map" to the name of the level to load. In the map you will be taken to, you must have an info_player_start, which will be the player's point of origin.

If you want to have a seamless transition, like in Half-Life when the 'Loading' message comes up, you need to do a bit of work.

First, you must have two maps with an end and start that match up exaclty. This will conceal the fact that you moved to another map when you actually moved to another map that looks the same! The easiest way to do this is make a hallway with a turn in it in Map 1 and a good ways behind the corner (until you can see the room on the other side) place your trigger_changelevel there. The trigger_changelevel is a brush with the trigger texture on it, so place it in a spot where the player is sure to pass through it. In the next map you'll only need to put the hallway, rather than the hall and the room behind it because all you can see is the hallway from the 2nd map.

Ok, now place an info_landmark in the same point of reference to the room in both maps, and give them both the same "targetname". Now, set the "map" in the trigger_changelevel to the name of the map to load next, and also set "landmark" to the "targetname" of the info_landmark that is in both maps. You MUST have a info_landmark and a trigger_changlevel in both maps! Even if you don't want the player to be able to go back, they still need to be there. To prevent the player from somehow going back, make the trigger_changlevel really small and up by the sky so the player can't get to it.

Another entity you can use is the trigger_transition. This is a really cool entity, because it allows NPC's and Monsters to follow you into the next map! The trigger_transition is a brush with the trigger texture on it and it's "targetname" is the same as the "targetname" of the info_landmark. What will happen is anything that is inside the trigger_transition will be taken to the next map, so this brush should probably be rather large.

Valid Keys/Values and Flags
    keyvalues
    targetname - name
    map - name of map to load next
    landmark - name of landmark in both levels that defines the same point in space
    changetarget - change target
    changedelay - delay before change target

    flags
      no intermission (1) - stops player from going back
      use only (2)
