Mapping Day of Defeat - by Nick Robins

DoD Flag Objectives

Most DoD maps use CTF or Control The Flags as the primary mission objective. This example will consist of five flags, the central flag will require two players five seconds to cap.

Entitys used:

  • dod_control_point
  • dod_capture_area
  • dod_control_point_master
  • dod_score_ent

Step one: Set up the scoring.

Select the point entity tool (shift-e) and select dod_score_ent from the drop down menu in the sidebar on the right. Place two of these entitys somewhere suitable (this entity has no visible presence in game so place them somewhere convenient for you, such as near where the central flag will be).

    tweaking:
points - this specifys points awarded for mission completion (in this example a map capture).

Step two: Adding the master.

Select the point entity tool (shift-e) and select dod_control_point_master from the drop down menu in the sidebar on the right. Place the entity somewhere suitable (this entity has no visible presence in game so place it somewhere convenient for you, such as near where the central flag will be).

    tweaking:
time between point gives - one of the functions of the dod_control_point_master is to award points to the teams for each of the flags they hold. The interval is set here.

Step three: Adding the flags


click to enlarge

Select the point entity tool (ctrl-e) and select dod_control_point from the drop down menu in the right-hand sidebar. Place five of these. One where you want each of the flags to appear.

Note: to prevent the flag from "levitating" make sure that the brush face forming the ground bisects the entity box of the dod_control_point (see picture above).

Group name is the only field required for the flag(s) to function, however the central flag (which is to be captured via an area) needs more fields adjusting).

This flag will still require the "can [team] touch?" fields setting even if it is to be inaccesible (like the central flag in dod_thunder) in order to make the capture area progress display correctly.

    tweaking:

  • control point name - the name that the players will see in-game eg; "Church"
  • time based points value - how many points the team holding the flag will receive for each time period specified in the dod_control_point_master
  • points to capturer - how many points the player who caps the flag is awarded
  • control point index - the location of the control point tell-tale in the players HUD. 1 is the far left, counting up to n which is the right hand end, -1 (default) is random placement.

Step four: Adding the capture area

Create a brush completely covered with the "aaatrigger" texture. place this brush where you want the capture area to be. To make the brush into a capture area select it then click "tools - tie to entity" (ctrl+t) then select dod_capture_area from the drop down list in object properties (alt+enter).

For testing purposes it might be convenient to change the "number required to cap" fields to 1.
