Mapping Day of Defeat - by Nick Robins

Object Objectives

dod_object and dod_object_goal combinations can be used for both primary and secondary objectives.

DoD v1.X

In DoD v1.X a dod_object_goal is no longer required to make a simple wall that can be demolished with TnT (dod_object) as all the settings required such as "objective group", "time delay", "trigger when tnt is set" can all be made directly on the func_breakable representing the wall. My original "object objective" tutorial on setting up demolishable walls can be found here. This tutorial will create a simple "Capture the Documents" primary objective for the allied team.

The difference between a primary and a secondary objective is the final target entity for the completion of the objective. For a primary objective the target entity will be a dod_score_ent (possibly via another entity such as a multi_manager) which will end the round when the objective is completed. For a secondary objective the target entity can be virtually anything depending on the desired result of completion.

Entitys to trigger on completion can be specified in the properties of either the object or the goal. For a primary objective (such as "capture the documents") the entity to trigger on completion should probably be named in the properties of the dod_object since this is the key item, this also allows you to specify mutiple "delivery sites".

dod_object_goal has no team specific function, if you wish to make team specific object objectives then the objects / goals should be separated into two objective groups (one for each team), and the team specific options set on the dod_object.

It is possible to have a mixture of team specific objects and non team specific objects in a single group, it is also possible to have some of the objects in a group specific to one team and others specific to the other team. Caution is advised when mixing teams in this way, especially in regard to how objective completion is triggered.

Entitys used:

  • dod_object
  • dod_object_goal
  • multi_manager
  • dod_score_ent

Optional

  • ambient_generic

See related documents for non DoD entity references

Step one: Place the dod_score_ent

Select the point entity tool (shift-e) and select dod_score_ent from the drop down menu in the sidebar on the right. Place the entity somewhere suitable (this entity has no visible presence in game so place it somewhere convenient for you). Select the dod_score_ent and open object properties (alt+enter) and give it a suitable name such as allies_win.

  

dod_score_ent (with the standard FGD) will automatically assign 10 points as the winners award. The "message" feild is the text that appears in large letters in the center of the screen when the objective is completed.

The "reset X" feilds are not usually used, they may be needed if you have a complex setup with multiple missions using masters, in order to make sure that things are correctly reset before they are disabled.

DO NOT set "end game" to yes, doing so will cause the server to exit when the entity is triggered.

Step two: Add a "controler"

Select the point entity tool (shift-e) and select multi_manager from the drop down menu in the sidebar on the right. Place the entity somewhere suitable (this entity has no visible presence in game so place it somewhere convenient for you, next to the dod_score_ent is good). Select the multi_manager and open object properties (alt+enter) and give it a suitable name such as allies_capture. Next click the button in the object properties window labled smartedit, this enables you to add keys to the entity. Click new key and enter these values:

this means that "allies_win" (the dod_score_env) will be immediately the multi_manager is fired. This function could be carried out by a trigger_relay but using a multi_manager allows other effects to be triggered as well, such as extra text (in addtion to the text provided by the dod_score_ent) and victory music.

Step three: Adding the target zone

Create a brush completely covered with the "aatrigger" texture. Place this brush covering the volume where you want the documents to be delivered

To make the brush into a target select it then click "tools - tie to entity" (ctrl+t) then select dod_object_goal from the drop down list in object properties (alt+enter). Other properties to be modified are:

  

By default dod_object_goal displays a TNT sprite if you are in the zone without the required object, a simple "allies target" sprite could be substituted if there is no "documents" sprite avaliable.

Step four: Adding the documents

Select the point entity tool (shift-e) and select dod_object from the drop down menu in the sidebar on the right. place the entity where you want it to appear.
Note: dod_object(s) are affected by gravity so if placed above the desired point, it will drop to the ground when the map starts.

Select the dod_object and open object properties (alt+enter).

This is now a functional mission, to create a similar mission for the axis all you have to do is replace "allies" with "axis" in all the entitys.

Optional

Adding victory music

Select the point entity tool (shift-e) and select ambient_generic from the drop down menu in the sidebar on the right. The music will apear to be coming from the location where you place the entity so putting it somewhere near the center of the map is probably best.

Next select the multi_manager and add a new key.
