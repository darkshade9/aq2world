Mapping Day of Defeat - by Nick Robins

Old Object Objectives

dod_object and dod_object_goal combinations can be used for both primary and secondary objectives.

DoD v1.0

In DoD v1.0 the process outlined here is no longer required to make a simple wall that can be demolished with TnT (dod_object) as all the settings required such as "objective group", "time delay", "trigger when tnt is set" can all be made directly on the func_breakable. This process will still work but requires more entitys.

The difference between a primary and a secondary objective is the final target entity for the completion of the objective. For a primary objective the target entity will be a dod_score_ent (possibly via another entity such as a multi_manager) which will end the round when the objective is completed. For a secondary objective the target entity can be virtually anything depending on the desired result of completion.

Entitys to trigger on completion can be specified in the properties of either the object or the goal. in this example the objective is to blow a wall up as a secondary objective. Since there can be multiple walls and bombs the completion target is specified in the properties of the dod_object_goal. For a primary objective (such as "capture the documents") the entity to trigger on completion should probably be named in the properties of the dod_object.

dod_object_goal has no team specific function, if you wish to make team specific object objectives then the objects / goals should be separated into two objective groups (one for each team), and the team specific options set on the dod_object.

It is possible to have a mixture of team specific objects and non team specific objects in a single group, it is also possible to have some of the objects in a group specific to one team and others specific to the other team. Caution is advised when mixing teams in this way, especially in regard to how objective completion is triggered.

entitys used:

  • dod_object
  • dod_object_goal
  • multi_manager
  • func_breakable

optional:

  • env_sprite

see related documents for non DoD entity references

Step one: Making the wall

Place the wall that is going to be blown up, this may consist of more than one brush depending on situation but for now make a single brush and select it. To make the brush breakable select "tools - tie to entity" (ctrl+t) then select func_breakable from the drop down list in object properties (alt+enter), then modify the properties of the entity as follows:

  

If multiple brushes are used they can either all be tied to the same entity or all the entitys making up a wall should have the same name.

Note: subsequent walls (in the same system) would be called wall_2, wall_3, etc.

The value of between 150 to 200 for explode magnitude is the size of the explosion that will accompany the wall breaking. This explosion is not required to remove the wall, it is the result of the wall breaking and will happen as long as the explode magnitude is greater than zero. Somewhere in the range 150 to 200 is large enough to be convincing for a demolition charge but not large enough to kill at unreasonable distances.

Selecting material type cinderblock will tell half-life to use the cinderblock gib model for the pieces of wall.

Step two: Add a "detonator"

Select the point entity tool (shift-e) and select multi_manager from the drop down menu in the sidebar on the right. Place the entity somewhere suitable (this entity has no visible presence in game so place it somewhere convenient for you, next to the wall is good). Select the multi_manager and open object properties (alt+enter) and give it a suitable name such as wall_1_det. Next click the button in the object properties window labelled smartedit, this enables you to add keys to the entity. Click new key and enter these values:

This means that "wall_1" (the func_breakable) will be triggered five seconds after the multi_manager is fired. This function could be carried out by a trigger_relay but using a multi_manager allows other effects to be triggered as well, such as text, sound or a visual count-down.

Step four: Adding the TNT

Select the point entity tool (shift-e) and select dod_object from the drop down menu in the sidebar on the right. Place the entity where you want it to appear. Note: dod_object(s) are affected by gravity so if placed above the desired point, it will drop to the ground when the map starts.

Select the dod_object and open object properties (alt+enter). the only field here that must be completed is the group field. the other properties, such as restricting to a team are usually used in a "proper" map but are not required here.

Once you have one tnt set up, you can create as many as you like by shift-dragging it.

Optional

Adding more walls

Each wall consists of:

  • dod_object_goal
  • multi_manager
  • func_breakable

In order to add more walls just repeat the process outlined in steps one to three above, renaming entitys (wall_1 => wall_2, wall_1_det => wall_2_det, etc) as appropriate. You can have more bombs (dod_object) than walls (dod_object_goal) or more walls than bombs, this is one of the advantages of the DoD objective group system.

Optional

Adding a visual (numeric) countdown

Don't worry too much about precise placement at first as the entity box size will change when you specify the sprite.
