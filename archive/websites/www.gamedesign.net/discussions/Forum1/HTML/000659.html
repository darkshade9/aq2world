You can't load a demo upon completion of a map (at least I don't think so, this has me thinking now...)
But you can load a demo or a series of demos, followed by the loading of a map if you wish. What is needed is a config file.

Like so:



alias demo1 "demomap mydemo1.dm2;set nextserver demo2"

alias demo2 "demomap mydemo2.dm2;set nextserver demo3"

alias demo3 "demomap mydemo3.dm2;set nextserver startgame"

alias startgame "map mymap"

demo1

This would load 3 demos in a series, and then load a map. A little editing would create a demo loop... which is what is done in DEFAULT.CFG in PAK0.PAK.

Now how to get this stuff loaded. You could create a config file called MYSTART.CFG and direct the user to exec the config from the console, or add the command at the end of the command line when the game is started. Or, if you are making your own pakfile, and your map will run in it's own game directory, you could include a copy of DEFAULT.CFG in there with the appropriate commands added, then they would load on startup automatically.
MD
