Introduction

First off, this SinED USER GUIDE will help you to get used to SinED, the best editor IMO. Hopefully by the time you finish reading this, you'll know what you're doing... if you don't, read it again. SinED is geared toward a 3 Button mouse, so you really should have one if you're going to do some serious mapping. Also, this is a development tool, so not all systems can run SinED. I have a P166 w/24 RAM and it runs pretty decent on it.

Second, I cannot offer an support for SinED or any of the Sin Tools. I'm busy working on my own stuff and barely have any spare time. This tutorial, along with the SinED Readme file, has all the information that you need if you just take the time to thoroughly read them.

Thanks to EutecTic from The Node for a lot of error fixes :)

Getting Started

Now you're ready to start up SinED. Click 4 times on sined.exe, it will open 2 sessions of SinED. One or both will crash. If both crash open SinEd one more time and it will work.

Arranging the Views

Below is the setup I like to use for my windows, I make my XY view as large as possible, which makes editing a lot easier. Keep in mind you only need to see one view at a time, so when you need to see the camera or texture picker you can just click on their top bar. Then when you need to see the XY again just click anywhere in it.

After you are comfortable with the views now it's time to get aquainted with all the cool features of SinED! First we'll go over what all of the buttons actually do. After that we'll be building a small map that should get you pointed in the right direction to start your own map.

Quick Buttons

  • Flip X - Flip a brush on the X-Axis
  • Flip Y - Flip a brush on the Y-Axis
  • Flip Z - Flip a brush on the Z-Axis
  • Rotate X - Rotate a brush 45 degrees on the X-Axis
  • Rotate Y - Rotate a brush 45 degrees on the Y-Axis

  • Select touching - Make a brush and any brush touching it will be selected
  • Select partial tall - Same as Select complete tall, but it doesn't select the full Z-Axis
  • Select inside - Make a brush and any brushes completely inside it will be selected
  • Make hollow - Make a brush hollow
  • CSG subtract - Subract the subtracted brush from it's surroundings
  • Wireframe - Makes the camera view display wireframes
  • Solid - Makes the camera view display solid colored brushes
  • Textured - Makes the camera view display textured brushes (Trilinear)
  • Tx Flip - Flip a texture on the X-Axis
  • Ty Flip - Flip a texture on the Y-Axis
  • Open SCR - Open the script file, or create one if it doesn't exist

BSP Menu

The Bsp Menu will not work as it was meant to run on a Unix machine. So, what I do is make a .bat file and put it in my "C:\tools\sin\bin" folder. Then in the .bat file I put 3 lines like this:

qbsp3 mapname.map
qvis3 mapname.bsp
qrad3 mapname.bsp
You can put whatever command lines you want like "qvis3 -fast mapname.bsp" or whatever you want.

If you are just testing your level, run qbsp3 alone. If you want to see what your lights look like you can run a full compile but do a "qvis3 -fast". For a final compile do a full compile with "qrad3 -extra".

Surface Inspector

This is the surface inspector, which can be accessed by pressing 'S'. It allows you to change all the texture attributes for any selected brushes. The Surface Inspector is the same as the Wide Surf Inspector, however the Mini Surf Inspector can only Strectch, Shift, Rotate textures.

Also, take note that anything that is a Surface Flag can be set to any number of faces on a brush, but a Content Flag must be set to entire brush.

Texture - Displays the current texture path/name
Surface Type - Select what the surface is made of, affects footstep sounds
Next Animation - Next texture to play in the animation
Animation Time - Ammount of time for animations
Horizontal Shift - Move the texture horizontally
Vertical Shift - Move the texture vertically
Horizontal Stretch - Stretch the texture horizontally (-1.00 mirrors it)
Vertical Stretch - Stretch the texture vertically (-1.00 mirrors it)
Rotate - Rotate the texture
Radiosity/Nonlit - Radiosity is the ammount of light given off. Nonlit is the ammount (0-1) of light a brush has, it will not emit light however
Radiosity Color - Click the color box to set the light color to be given off
Direct Value/Angle - Direct Value is the intensity of the light emitted by the brush face. You have to set both the 'light' and 'prelit' surf flags plus a value in this field for a brush face to emit light at all. Depending on the radiosity color, values between 1000 and 10000 are common. A darker radiated color needs more intensity. Angle sets the "cone" angle of the light emitted by the brush face (0 - 180).

Translucent/Group - The group name is simply the brush face's name (like it's "targetname" if you will). If , for example, you set a brushe's face group to "face1", this is useful for 2 things. First, you will be able to refer to it in a script as @face1 to control it's texture translation parameters and wave effects in the case of liquid brushes (water, lava or slime). More on this below. Also, it's used when for example you make a console entity and want the console display to appear on this brush face, you set the group field to the same name as the console entity's 'consolename' key. Please note that your brush HAS to be either the console entity itself or a func_scriptobject entity for this to work. Several brush faces can have the same group name if you want to control the textures on all those simultaneously or if you want the same console display to be shown on several brush faces. Please note that for the 'Translucent' value to have any effect, the 'translucent' content flag also has to be set.

Friction/Restitution - Friction sets how fast the player can move across the brushes or in a liquid how high the waves will swell up and down. Restitution will make the player bounce on a brush, but with a liquid brush it sets the speed at which those waves move.

Translation Magnitude/Angle - Magnitude sets the speed in pixels/sec of the translation (scrolling) of the texture on a brush face. Angle sets the direction in which the texture will scroll on the brush face. Here again, by setting the 'group' field, you can use simple script commands to make the translation accelerate, slow down, stop, change direction, restart again, etc. Please note that the 'translate' surface flag must be set for this to work.

Width/Height - Current textures size in pixels

Texture Picker

This is the texture picker, which can be accessed by pressing 'T'. To see any textures you must have them extracted from the pak files and be in "C:\tools\sin\base\textures" or wherever your sample.sed file points to for textures. Next click the textures menu and load a group by selecting it from the menu. Now you can see that all of your textures are there for you to use.

At the top of the texture picker is the path/name of the texture and it's size in pixels. Also, notice that textures that have been used are outlined in green and the current texture is outlined in red. To move up an down just hold down the right mouse button in the texture picker and move up and down.

To apply a texture you must select a face (CRTL+SHIFT+LEFT MOUSE) or a brush (SHIFT+LEFT MOUSE) and click on the texture in the texture picker. If you already have used a texture and you have a 3 Button mouse, you can select a brush and click on the brush that has the texture you want to use with the middle button.
