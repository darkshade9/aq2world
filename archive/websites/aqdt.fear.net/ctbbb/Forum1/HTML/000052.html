Monk, since I never explained it (though I'm sure hal could have told you this in about two seconds) here's the skinny on the match.ini.

Match.ini is action.ini without team name, and it only should get used in the place of action.ini when matchplay is set to 1. Team names default to team 1 and team 2 and the skins default to the ogl-specific skins, and the only purpose of the match.ini is to create the appropriate set of eligible maps to enable voting on legal maps only (so paste the current ogl maplist in and you're set).

Basically it allows a server admin to avoid needing to give out lrcon or rcon in order for teams to play a match. From what I gather in my hermit's perch, two teams can now secure a server based on captain agreement, so this now allows a server admin to set up legal voting while still retaining the public server setting used in the action.ini. Of course, hal may have redone map voting at some point to replace the vote command from TE 1.2a, so this could be useless now.
